## Why

Phase 1 の機能 (商品の登録・一覧・削除) は動作するが、見た目が素朴で UX が低い。具体的にはボタンがボタンに見えず、モーダルダイアログのテキストが読みづらい、カードのレイアウトに視覚的階層がない。最終的には旧プロダクト (specs/old-product.md) のデザインに揃えるが、その前段階として最小限のスタイル整備を行う。

## What Changes

- **ボタン** — 視覚的にボタンと認識できる色・パディング・ホバー状態を付与 (主アクション/副アクション/破壊的アクションの色分け)
- **モーダル** — タイトル、ラベル、入力欄の余白とコントラストを整え、可読性を向上
- **カード** — 商品名とカテゴリの視覚階層を整理 (カテゴリをバッジ風に)、削除ボタンの位置と見た目を整える
- **ヘッダー** — 旧プロダクトと同じ teal グラデーション (`linear-gradient(141deg, #009e6c, #00d1b2, #00e7eb)`) のヘッダーをページ上部に追加し、タイトル "Pantry Panel" を表示
- **共通** — ページの全体レイアウト (max-width, padding) を整えて、空状態・エラー状態の見た目も統一

データモデル、API、追加機能の変更は **なし**。Tailwind のクラスのみで完結する。

## Impact

- **影響ファイル**:
  - `frontend/src/app/stock-items/page.tsx` (ヘッダー、レイアウト、空状態)
  - `frontend/src/components/CreateItemModal.tsx` (モーダル全体のスタイル)
  - `frontend/src/components/ItemCard.tsx` (カード、カテゴリバッジ、削除ボタン)
  - `frontend/src/app/layout.tsx` (必要であれば共通レイアウト)
- **依存関係**: 追加なし。既存の Tailwind CSS のみで実装
- **テスト**: 既存のユニットテストは振る舞いベースなので、スタイル変更による影響は最小。アクセシブルロール・ラベルが変わらない限り pass する想定
- **動作確認**: ローカル起動して目視確認 (旧プロダクトのスクリーンショットや見た目の参照は user 側で実施)
